# Contributing

Thank you for contributing to Elcorp Namibia!

- Use the main branch for releases and create feature branches for development.
- Open pull requests against `main`.
- All PRs must pass CI (linting, tests, security scan).
- Use the pre-commit hooks provided; run `pre-commit install`.
- Follow the code style (Black + Ruff).
- For large changes, open an issue first to discuss the design.

PR checklist:

- [ ] Tests added/updated
- [ ] Linting passed
- [ ] Security/secret checks passed
- [ ] Documentation updated (if applicable)
